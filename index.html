What is Front End Development?

My Definition

Technical implementation and best practices for user facing page elements.





What is it not?

  • Server side development

  • SEO from a content strategy standpoint

  • Marketing

  • Social-media douchbaggery

But it is...

  • Interacting with server side code and writing server side views.

  • SEO from a technical practices standpoint

  • Landing page best practices

  • Implementing social widgets and required meta-data

We are entering the
JavaScript age

  • I. 1991-1999: The HTML Age.

  • II. 2000-2009: The LAMP (Linux/Apache/MySQL/PHP).

  • III. 2010-??: The JavaScript age

  • The JavaScript age is about event streams. Modern web pages are not pages, they are event-driven applications through which information moves.

  • The JavaScript age brings us closer to a web that is not a global digital library, but a global digital nervous systen, whose implications we are only beginning to apprehend. (Source)

CSS3

  • Rounded corners, Shadows, Transforms, multiple backgrounds

  • New selectors

  • New layout options (columns, tables)

JavaScript

  • Local Storage

  • Offline Storage

  • SVG

  • New Events (touch, orientation, acceleration)

  • Geolocation

5 Tenets of modern front end development

#1 Semantics

Give your code meaning and the Google will reward you.

#2 Progressive
Enhancement

Start with a workable baseline, add features as supported.

www.DoWebsitesNeedToLookExactly-TheSameInEveryBrowser.com

#3 In jQuery
we trust

Learn jQuery first, then JavaScript. Understand your selectors and how to use plugins and you can fake it for a long time

#4 Capabilities,
not browsers

Don't look at the user agent, this stuff changes way too quickly.

Use modernizr or the like and make decisions based on capabilities.

#5 It's no longer just the desktop

Mobile and Tablet browsers are taking over. This year double digits, in a couple years 50% (Made up number).
